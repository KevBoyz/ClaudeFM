**Data:** 2026-05-26

## 1. Visão Geral

Interface HTML/CSS/JS renderizada via `pywebview`. Sem framework JS (vanilla). SPA com roteamento client-side via `router.js`. Todo estado UI vive em memória no JS — nenhum `localStorage`, nenhuma chamada direta ao DB.

**Idioma da interface:** inglês.

## 5. Roteamento (router.js)

SPA com páginas renderizadas dinamicamente em `#main-content`.

```javascript
router.navigate(route, params)   // troca página e atualiza histórico
router.back()                    // volta na pilha de histórico
router.current()                 // retorna rota ativa
```

| Rota | Página |
|------|--------|
| `home` | Home — biblioteca local |
| `artists` | Artists — grid de artistas |
| `albums` | Albums — grid de álbuns |
| `artist/:id` | Artist detail (biblioteca local) |
| `album/:id` | Album detail (biblioteca local) |
| `lastfm/artist/:name` | Last.fm Artist page |
| `lastfm/album/:artist/:title` | Last.fm Album page |
| `playlists` | Playlists |
| `playlist/:id` | Playlist detail |
| `downloads` | Downloads |
| `settings` | Settings |

## 6. api.js — Bridge JS ↔ Python

### Chamadas para Python

```javascript
// Biblioteca
api.get_library(sort_by, filter_format)    // → [{track}]
api.search_local(query)                    // → [{track}]
api.get_track(track_id)                    // → {track}
api.get_artists()                          // → [{artist, track_count}]
api.get_albums()                           // → [{album, artist, track_count}]
api.get_tracks_by_artist(artist)           // → [{track}]
api.get_tracks_by_album(album, artist)     // → [{track}]

// Last.fm
api.search_lastfm(query, type)             // → [{result}] — type: 'artist'|'track'|'album'
api.get_artist_top_tracks(artist)          // → [{track}]
api.get_album_tracks(album, artist)        // → [{track}]

// Downloads
api.queue_download(track_id)              // → {ok}
api.check_connectivity()                  // → {ok} | {error}

// Playback
api.play(track_id, context_ids)           // → {ok}  — context_ids define fila
api.pause()                               // → {ok}
api.resume()                              // → {ok}
api.next_track()                          // → {ok}
api.prev_track()                          // → {ok}
api.seek(position)                        // → {ok}
api.get_position()                        // → {position: float}
api.set_volume(level)                     // → {ok}  — 0.0–1.0

// Playlists
api.get_playlists()                       // → [{playlist}]
api.get_playlist_tracks(playlist_id)      // → [{track}]
api.create_playlist(name)                 // → {playlist}
api.rename_playlist(playlist_id, name)    // → {ok}
api.delete_playlist(playlist_id)          // → {ok}
api.add_to_playlist(playlist_id, track_id)    // → {ok}
api.remove_from_playlist(playlist_id, track_id) // → {ok}

// Letras
api.fetch_lyrics(track_id)               // → {ok} | {error}
api.fetch_missing_lyrics()               // → {ok}  — dispara em background
api.get_lyrics(track_id)                 // → {lyrics: string|null}

// Settings
api.get_settings()                       // → {key: value}
api.set_setting(key, value)              // → {ok}
api.check_lastfm_connection()            // → {ok} | {error}
```

Todas as chamadas retornam Promise. Respostas são JSON strings (`JSON.parse` automático em `api.js`).

### Padrão de response

```json
{"status": "ok", "data": {...}}
{"status": "error", "message": "..."}
```

### Eventos push (Python → JS)

`api.js` registra `window.onEvent` e despacha por `type`:

| Event type | Payload | Handler |
|---|---|---|
| `download_progress` | `{track_id, percent}` | `downloads.onProgress` |
| `download_complete` | `{track_id}` | `downloads.onComplete` |
| `download_error` | `{track_id, message}` | `downloads.onError` |
| `library_scan_complete` | `{added, missing}` | `home.onScanComplete` |
| `playback_ended` | `{}` | `player.onEnded` |
| `restore_player` | `{track_id, position}` | `player.onRestore` |
| `lyrics_progress` | `{track_id, status}` | `lyrics.onProgress` |
| `lyrics_fetch_complete` | `{synchronized, plain_text, instrumental, not_found, errors}` | `lyrics.onFetchComplete` |

## 7. Componentes

### 7.1 Topbar

```
[≡] ClaudeFM    [Home] [Artists] [Albums] [Playlists] [Downloads] [Settings]    [⬇ 2]
```

- `[≡]` toggle sidebar. Ativo = sidebar visível.
- Nav links: highlight na rota ativa.
- `[⬇ N]`: badge com downloads ativos. Click abre `download-panel.html` como dropdown. Se `N = 0`, badge oculto.

### 7.2 Sidebar

```
┌──────────────────────────┐
│ [Search input...    ] [🔍]│
│ [Last.fm] [Local]         │
│ ○ Artist  ○ Track  ○ Album│
├──────────────────────────┤
│ (resultados)              │
│  card                     │
│  card                     │
│  card                     │
└──────────────────────────┘
```

- Busca dispara SOMENTE ao clicar `🔍` ou pressionar Enter. Sem debounce.
- Toggle Last.fm / Local: muda fonte da busca.
- Radio buttons: `Artist`, `Track`, `Album`.
- Resultados: lista compacta até `search_results_limit` itens com scroll interno.
- Controls (input + toggles + radios) fixos no topo — nunca fazem scroll.

**Search result cards (sidebar):**

| Tipo | Conteúdo | Click | Hover button |
|------|----------|-------|--------------|
| Artist | Nome + ícone | Abre `lastfm/artist/:name` | — |
| Album | Título + artista | Abre `lastfm/album/:artist/:title` | `⬇` download all |
| Track (Last.fm) | Título + artista | Abre track detail / highlight | `⬇` download |
| Track (local) | Título + artista | Play imediato, define fila | — |

**Estados do botão `⬇` inline:**

| Condição | Estado |
|----------|--------|
| Não baixada | `⬇` enabled |
| `download_status='completed'` + `file_status='available'` | `✓` disabled |
| `file_status='missing'` ou `'corrupted'` | `⬇` enabled |
| `download_status='downloading'` | spinner disabled |
| `download_status='failed'` | `⬇` retry enabled |

Click → spinner imediato (optimistic UI); contador topbar incrementa.

### 7.3 Player Bar

```
[capa 48px] [título · artista]      [⏮] [⏯] [⏭]    [────────────] 🔊 [♪]
            [lyrics_status icon]    [  progress bar  ]    vol
```

- Capa: `placeholder.png` universal (48×48px).
- Título + artista: truncado com `text-overflow: ellipsis`.
- `lyrics_status` icon: indicador visual do status da letra da faixa atual:
  - `🎵` = `synchronized` ou `plain_text` — clicável, abre lyrics panel
  - `—` = `not_fetched` ou `not_found` — não clicável (ou clique dispara fetch manual)
  - `🎸` = `instrumental` — não clicável
- `⏯`: alterna play/pause.
- Progress bar: clicável para seek. Mostra posição atual / duração total.
- Volume: slider horizontal. Nível salvo em `settings.player_volume`.
- `[♪]` botão: abre/fecha lyrics panel (overlay sobre main content).


- Se `plain_text`: exibe texto puro sem destaque.
- Se `instrumental`: exibe "Instrumental".
- Se `not_found`: exibe "Lyrics not found" + botão retry.
- Se `not_fetched`: exibe "Lyrics not fetched yet" + botão fetch.

## 9. Estado Global (JS)

### player.js

```javascript
player.state = {
  track: null,          // objeto track atual
  playing: false,
  position: 0,          // segundos
  duration: 0,
  volume: 1.0,
  queue: [],            // array de track_ids
  queueIndex: 0,
  queueName: '',        // nome do contexto

player.play(track_id, context_ids, queue_name)
player.pause()
player.resume()
player.next()
player.prev()
player.seek(position)
player.setVolume(level)
player.onEnded()        // handler do evento push
player.onRestore(track_id, position)
```

Tick de posição: `setInterval` a cada 1s chama `api.get_position()` e atualiza progress bar.

### downloads.js

```javascript
downloads.active = {}   // track_id → {track, percent}
downloads.history = []  // [{track, status, error}]

downloads.queue(track_id)
downloads.onProgress(track_id, percent)
downloads.onComplete(track_id)
downloads.onError(track_id, message)
downloads.activeCount()
```

### lyrics.js

```javascript
lyrics.state = {
  track_id: null,
  text: null,           // string bruta (LRC ou plain)
```

## 10. Comportamentos Transversais

### Toasts

Sistema de notificações leve (sem biblioteca externa):

| Evento | Toast |
|--------|-------|
| Download complete | "Downloaded: {title}" (success, 3s) |
| Download error | "Download failed: {title}" (error, 5s) |
| Library scan complete | "Library updated: {added} added, {missing} missing" (info, 4s) |
| Last.fm connection success | "Connected to Last.fm" (success, 3s) |
| Last.fm connection fail | "Last.fm connection failed" (error, 5s) |
| Lyrics fetch complete (bulk) | "Lyrics: {found} found, {not_found} not found, {errors} errors" (info, 5s) |
| Settings saved | "Settings saved" (success, 2s) |

Toast: canto inferior direito, acima do player bar. Máximo 3 simultâneos (FIFO). Animação fade-in/out.

### Loading States

- Busca: spinner no botão da lupa enquanto aguarda resposta.
- Páginas: skeleton loader (cinza animado) antes dos dados chegarem.
- Download: spinner no botão do card imediatamente ao click (optimistic).
- Lyrics fetch: spinner no botão do panel durante o fetch.

### Empty States

| Página/Contexto | Mensagem |
|----------------|----------|
| Library vazia | "No tracks yet. Search for music or configure a folder in Settings." |
| Artists vazio | "No artists in your library." |
| Albums vazio | "No albums in your library." |
| Playlist vazia | "This playlist is empty. Right-click any track to add it." |
| Search sem resultado | "No results for "{query}"." |
| Lyrics not found | "No lyrics found for this track." |
| Player sem faixa | "Nothing playing." |

### Confirmações

Ações destrutivas exibem modal de confirmação simples (não `window.confirm`):
- Deletar playlist: "Delete "{name}"? This cannot be undone."
- Remover da biblioteca: "Remove "{title}" from library? The file will not be deleted."

### Erros de Conectividade

Antes de qualquer operação que requer internet (Last.fm search, download):
- `api.check_connectivity()` → se erro, exibe dialog e aborta operação.
- Sem diálogo para operações offline (playback, biblioteca local).

### Right-click global

`context-menu.js` escuta `contextmenu` em todo o documento. Fecha ao clicar fora ou `Escape`. Posição relativa ao cursor, com ajuste para não ultrapassar viewport.

## 11. Integração de Letras

### Fluxo automático (pós-download)

1. `api.queue_download(track_id)` é chamado (JS → Python).
2. Backend baixa o arquivo + fetch de letras automaticamente (se `auto_fetch_lyrics=true`).
3. Evento `download_complete` chega → `downloads.onComplete` → atualiza UI do card.
4. Não há evento separado para letras nesse fluxo — status é atualizado silenciosamente no arquivo.

### Fluxo manual (by track)

1. Usuário right-click → "Fetch lyrics" (visível se `not_fetched` ou `not_found`).
2. Chama `api.fetch_lyrics(track_id)`.
3. Spinner no menu enquanto aguarda.
4. Response → atualiza `lyrics_status` no card + player bar icon.

### Fluxo em lote

1. Via Settings ou botão dedicado → `api.fetch_missing_lyrics()`.
2. Eventos `lyrics_progress` chegam → `lyrics.onProgress` → atualiza ícone nos cards afetados.
3. Evento `lyrics_fetch_complete` → toast com resumo.

### Lyrics Panel — sincronização

Se `lyrics_status = 'synchronized'`:
- `lyrics.js` parseia o texto LRC linha a linha: `[MM:SS.xx] texto`.
- A cada tick do player (1s), compara `player.state.position` com timestamps.
- Linha atual ganha classe `active`, panel faz scroll automático para ela.

## 12. Acessibilidade e UX Mínima

- Focus visible em todos os elementos interativos.
- Keyboard: `Space` play/pause, `←/→` seek ±5s, `N` próxima, `P` anterior — quando focus não está em input.
- Cursor `pointer` em todos os elementos clicáveis.
- Desabilitar botões durante operações em andamento (não apenas visual — `disabled` attribute).
- Scrollbars: estilizadas para combinar com o tema (thin, arredondadas).
