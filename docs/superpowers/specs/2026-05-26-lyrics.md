# Spec: Lyrics Feature

Fetch lyrics from LRCLIB, embed directly into audio file metadata tags.

## Decisions

| # | Decision | Choice |
|---|----------|--------|
| 1 | When to fetch | Auto post-download (default). Configurable. On-demand bulk fetch available. |
| 2 | Library | `lrcup` â€” `LRCLib` (API client) + `AudioFile` (mutagen wrapper) |
| 3 | Format | Synced LRC preferred; plain text fallback. Stored in file tag only. Raw LRC string returned to frontend â€” no backend parsing. |
| 4 | DB tracking | `lyrics_status` column â€” 6 values (see below) |
| 5 | Lookup strategy | `get()` first â†’ fallback `search()` â†’ first result |
| 6 | Network errors | Keep `not_fetched`; 404/empty â†’ `not_found`; unsupported format â†’ `not_supported` |
| 7 | Service location | `src/services/lrclib_service.py` â€” class `LRCLibService(conn)` |
| 8 | API methods | `fetch_lyrics`, `fetch_missing_lyrics`, `get_lyrics` |
| 9 | Config | `auto_fetch_lyrics = true`, `player_volume = 1.0` in DEFAULTS |
| 10 | Orchestration | `api.py` passes `on_complete` hook (`fetch_and_embed_async`) to `queue_download()` |
| 11 | Bulk fetch threading | Background daemon thread, protected by `threading.Event` against double execution |
| 12 | Bulk fetch events | `lyrics_progress` per track + `lyrics_fetch_complete` with summary |
| 13 | `get_lyrics` source | Reads from audio file tag via `lrcup.AudioFile`; returns `{lyrics, lyrics_status}` |
| 14 | DB migration | Add column directly to `CREATE TABLE tracks` â€” no existing DB |
| 15 | `on_complete` threading | `fetch_and_embed_async` spawns own daemon thread â€” download worker not blocked |
| 16 | `fetch_and_embed` return | Returns status string on completion, `None` on early return (no track / no file_path) |

## 2. Database Schema Change

In `init_db` â†’ `CREATE TABLE tracks`, add column:

```sql
lyrics_status  TEXT NOT NULL DEFAULT 'not_fetched'
```

Valid values: `not_fetched`, `not_found`, `not_supported`, `instrumental`, `plain_text`, `synchronized`

- `not_fetched` â€” never attempted or failed with network error (retryable)
- `not_found` â€” LRCLIB returned no results (retryable manually)
- `not_supported` â€” file format not supported by lrcup (no retry)
- `instrumental` â€” LRCLIB confirmed track is instrumental
- `plain_text` â€” plain lyrics embedded in file tag
- `synchronized` â€” LRC lyrics with timestamps embedded in file tag

Add new CRUD functions to `database.py`:

```python
def update_lyrics_status(conn, track_id: int, status: str) -> None
def get_tracks_without_lyrics(conn) -> list[Track]  # lyrics_status = 'not_fetched' only
```

`_row_to_track` must include `lyrics_status` in the returned `Track` object.
Add `lyrics_status: str = "not_fetched"` to `Track` model.

## 4. `src/services/lrclib_service.py`

### Class: `LRCLibService(conn)`

```python
class LRCLibService:
    def __init__(self, conn: sqlite3.Connection) -> None
    def fetch_and_embed(self, track_id: int) -> str | None   # returns status or None
    def fetch_and_embed_async(self, track_id: int) -> None   # fire-and-forget daemon thread
    def fetch_missing_lyrics(self) -> None                   # starts background thread
    def get_lyrics(self, track_id: int) -> dict | None
```

`__init__` sets `self._running = threading.Event()`.

### `fetch_and_embed(track_id)` â†’ `str | None`

Returns the final `lyrics_status` string, or `None` if early exit (no DB update performed).

1. Load track from DB. If not found or `file_path` is None â†’ return `None`.
2. Try `LRCLib().get(track=title, artist=artist, album=track.album or "", duration=duration)`:
   - Skip `get()` if `duration` is None â†’ go directly to step 3.
   - On network/HTTP error â†’ log, `update_lyrics_status(conn, track_id, "not_fetched")`, return `"not_fetched"`.
3. If `get()` returns None â†’ try `LRCLib().search(track=title, artist=artist)` â†’ take `results[0]` if any.
   - On network/HTTP error â†’ log, `update_lyrics_status(conn, track_id, "not_fetched")`, return `"not_fetched"`.
4. If no result after both â†’ `update_lyrics_status(conn, track_id, "not_found")`, return `"not_found"`.
5. If `result.instrumental` â†’ `update_lyrics_status(conn, track_id, "instrumental")`, return `"instrumental"`.
6. Determine lyrics and state:
   - `syncedLyrics` not None â†’ `lyrics = syncedLyrics`, `state = "synced"`, `status = "synchronized"`
   - else `plainLyrics` not None â†’ `lyrics = plainLyrics`, `state = "unsynced"`, `status = "plain_text"`
   - else â†’ `update_lyrics_status(conn, track_id, "not_found")`, return `"not_found"`.
7. `AudioFile(Path(track.file_path)).set_lyrics(state=state, lyrics=lyrics)`
   - On `UnsupportedSuffix` â†’ log, `update_lyrics_status(conn, track_id, "not_supported")`, return `"not_supported"`.
8. `update_lyrics_status(conn, track_id, status)`, return `status`.

### `fetch_and_embed_async(track_id)`

Fire-and-forget. Spawns a daemon thread that calls `fetch_and_embed(track_id)`. Returns immediately.

```python
def fetch_and_embed_async(self, track_id: int) -> None:
    threading.Thread(target=self.fetch_and_embed, args=(track_id,), daemon=True).start()
```

Used as the `on_complete` hook in `api.py` â€” keeps download worker thread unblocked.

### `fetch_missing_lyrics()`

Starts a background daemon thread. If already running (`self._running.is_set()`), returns immediately without starting a second thread.

```python
def fetch_missing_lyrics(self) -> None:
    if not self._running.is_set():
        self._running.set()
        threading.Thread(target=self._run_batch, daemon=True).start()
```

`_run_batch()`:

1. Loads all tracks with `lyrics_status = "not_fetched"` from DB.
2. Initializes counters: `synchronized=0, plain_text=0, instrumental=0, not_found=0, not_supported=0, errors=0`.
3. For each track, wrapped in `try/except Exception`:
   - Calls `fetch_and_embed(track.id)` â†’ gets `status`.
   - On unexpected exception â†’ log, `status = "not_fetched"`, increment `errors`.
   - Increments the matching counter (if `status = "not_fetched"` from network error â†’ increment `errors`).
   - Emits `event_bus.emit("lyrics_progress", {"track_id": track.id, "status": status})`.
4. Calls `self._running.clear()`.
5. Emits:
   ```python
   event_bus.emit("lyrics_fetch_complete", {
       "synchronized": int,
       "plain_text": int,
       "instrumental": int,
       "not_found": int,
       "not_supported": int,
       "errors": int,
   })
   ```

### `get_lyrics(track_id)` â†’ `dict | None`

1. Load track from DB. If not found or `file_path` is None â†’ return `None`.
2. `text = AudioFile(Path(track.file_path)).get_lyrics()` â€” raw string (LRC or plain) or None.
3. On any exception â†’ log, return `None`.
4. Return `{"lyrics": text, "lyrics_status": track.lyrics_status}`.

### Lazy initializer

```python
def _get_lrclib(self) -> LRCLibService:
    if self._lrclib is None:
        self._lrclib = LRCLibService(self._conn)
    return self._lrclib
```

### `queue_download` wiring

Passes `fetch_and_embed_async` (not `fetch_and_embed`) â€” download worker thread is not blocked by lyrics network calls.

```python
def queue_download(self, track_id: int) -> str:
    auto = get_setting(self._conn, "auto_fetch_lyrics") == "true"
    hook = self._get_lrclib().fetch_and_embed_async if auto else None
    self._get_youtube().queue_download(track_id, on_complete=hook)
    return self._ok()
```

## 8. Frontend Notes

### `not_supported` status UI

- Player bar icon: same `â€”` as `not_found` â€” not clickable.
- Context menu: "Fetch lyrics" is NOT shown for `not_supported` (retrying would always fail).
- Lyrics panel: display "Lyrics not available for this format."

### `get_lyrics` response rendering

Frontend uses `lyrics_status` from response to decide rendering:
- `synchronized` â†’ parse LRC timestamps, sync highlight with player position.
- `plain_text` â†’ display raw text, no timestamps.
- `instrumental` â†’ display "Instrumental".
- `not_found` / `not_supported` â†’ display message + retry button (except `not_supported`).
